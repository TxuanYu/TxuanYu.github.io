而npm install會修改node_modules資料夾內部，npm install hexo-deployer-git --save會更新package.json、package-lock.json這兩個檔案。

當我們執行hexo s，會生成db.json並修改package.json :

hexo cl 後刪除db.json、public資料夾；hexo g (根據source檔案)生成public資料夾、db.json；
hexo d (根據public檔案)修改.deploy_.git。

由上述敘述可知，要修改檔案必須從source裡修改(如: 刪除文章時)，因為public是依據source生成；而deploy_.git是依據public生成。

blog（根目錄）

├─ .deploy_git # 部署至github的相關檔案

├─ node_modules # hexo的相關檔案

├─ public　# 暫存資料夾(根據source檔案生成)

├─ scaffolds # 初始化檔案

　　├─ _draft

　　├─ _page

　　├─ _post

├─ source　# push 到網站上的檔案

　　├─ _drafts　# 尚未發布的文章

　　├─ _posts　# 已發佈的文章(若需要刪除文章，直接從這邊刪除即可)

├─ themes　# 主題資料夾

　　└─ xxxx　# 自己下載的主題名稱

└─ _config.yml　# 部落格設定

注意事項

  1. 將部落格部署到github後，會將成程式碼的執行檔上傳到倉庫(repository)中，這個執行檔並不能直接更改樣式或文章，只能透過原始檔來進行更變，因此本地端的資料夾非常重要，要時常備份!

  2. 為了避免單一文章篇幅過長，可在指定位置加入<!–more–>，就能透過點選繼續閱讀來觀看完整內容。

  3. 文章在Hexo的資料夾架構，都是放在source資料夾底下，而其中

    • _discarded : 刪除的文章
    • _drafts : 未發布的文章
    • _posts : 已發佈的文章

  4. 如果要修改文章的網址連結的話，要直接修改黨名。
